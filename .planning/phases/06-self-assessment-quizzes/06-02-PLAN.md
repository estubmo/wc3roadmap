<objective>
Lay the persistence foundation for quiz-driven mastery: widen the `source` enum to include `"quiz"` (D-12), and add a separate `quizProgress` Drizzle table that records the forward-SRS signals a future spaced-repetition engine needs — `passed`, `lastAttemptAt`, `attemptCount`, and `lapseCount` (D-07/D-08). Then apply the new table to Neon via the mandatory schema push.

Purpose: `nodeProgress` is the mastery-state record (what the graph shows); `quizProgress` is the SRS tracking record (what a future scheduler needs). Keeping them separate avoids coupling the mastery display to the quiz retention model. `lapseCount` is added now because it is impossible to reconstruct from `attemptCount` retroactively.
Output: extended `source` enum, the `quizProgress` table + relations, and the live table in Neon.
</objective>
