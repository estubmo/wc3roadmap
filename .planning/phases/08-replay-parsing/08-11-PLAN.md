<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → uploadReplay | Untrusted FormData `.w3g` binary crosses into the parser |
| client → pull/read fns | Authenticated request; no ownership token accepted |
| server → w3champions | Outbound gameId download (via 08-08 guard) |
| server → database | Monotonic-max mastery write + gameId cache write |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-11a | Elevation of Privilege (IDOR) | all nodeProgress/cache writes | high | mitigate | Every write/read keyed by `context.principal.id` (ADR 007); no userId/gameId ownership channel |
| T-08-11b | Tampering | source/patchId/buildNumber stamping | high | mitigate | All server-stamped hardcoded/derived; never read from FormData or upstream JSON (ADR 009) |
| T-08-11c | Tampering | monotonic-max downgrade bypass | high | mitigate | Atomic `setWhere` ordinal guard makes downgrade impossible at SQL level; not a check-then-write race |
| T-08-11d | Denial of Service | oversized/malformed upload | medium | mitigate | Size cap (ADR 011) enforced before parse; `parseReplay` try/catch → `parse-failed` bucket; zero writes on failure |
| T-08-11e | Information Disclosure | upstream/parse error strings | low | mitigate | Only opaque `ReplayStatus` buckets returned; no upstream/internal strings surfaced (T-07-06c) |
| T-08-11f | Spoofing | wrong-player attribution | medium | mitigate | BattleTag→slot match normalized (Pitfall 6); no match → `no-player-match`, zero advancement (D-14) |
</threat_model>
