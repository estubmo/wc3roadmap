## Three-Gold Visual Hierarchy (F-01 Resolution)

The CONTEXT identified three simultaneous uses of rune-gold as a design risk (F-01). Contract:

| Gold use | Token | Shape | Motion | Intensity |
|----------|-------|-------|--------|-----------|
| Mastered node | `rune-500` | Fill area (whole node) + solid border + strong glow | None — static | Highest — dominates |
| In-progress ring | `rune-600` | Border-only (ring, no fill) + soft ambient glow | None — static | Medium — perimeter |
| Edge chain highlight | `rune-400` | Stroke on edge lines only | Motion spring 300ms enter/exit | Lowest intensity, kinetic |

Differentiated by three orthogonal axes: (1) area vs perimeter vs line, (2) darker vs mid vs lighter rune stop, (3) static vs animated. All three can coexist on screen without muddying — a mastered node with an in-progress prerequisite connected by a highlighted edge reads as three distinct states.

## Node Visual Language

### Node Anatomy

Each node face renders (top-to-bottom, left-to-right):
1. **Type icon** — 16×16px lucide icon, top-left corner
2. **Mastery badge** — top-right corner (absent when untouched)
3. **Title** — 15px/600, max 2 lines, ellipsis overflow
4. **Difficulty dots** — bottom-left, 3-dot row, filled dots indicate level

Node face dimensions: **160px wide × 80px tall** (fixed — `nodesDraggable={false}` reduces event overhead per CLAUDE.md performance guidance).

### Node Shapes by Type (D-06)

Shapes must be identifiable at the minimum zoom level where icon glyphs blur (~50%).

| nodeType | Border radius | Clip / Shape | Rationale |
|----------|---------------|--------------|-----------|
| `mechanical` | 4px | Sharp rectangular corners | Precision, timing, mechanical clarity |
| `conceptual` | 16px | Rounded rectangle | Ideas, understanding, open-ended knowledge |

### Node Icons by Type (D-04)

From lucide-react:

| nodeType | Icon | aria-label |
|----------|------|------------|
| `mechanical` | `Sword` | "Mechanic node" |
| `conceptual` | `BookOpen` | "Conceptual node" |

### Mastery State Encoding (D-05)

Two simultaneous encodings: fill+glow progression AND status badge. Both required.

**Untouched:**
- Background: `--color-obsidian-800` (#1c1c25)
- Border: 1px solid `--color-obsidian-600` (#34343f)
- Glow: none
- Badge: none
- Opacity on canvas: 100% (pathway) or 20% (non-pathway, dimmed)

**In-progress ("Learning"):**
- Background: `--color-obsidian-800` (#1c1c25)
- Border: 2px solid `--color-rune-600` (#b9852c)
- Glow: `box-shadow: 0 0 8px 1px color-mix(in oklab, var(--color-rune-600) 30%, transparent)`
- Badge: small pill, `--color-rune-600` background, `--color-rune-300` text, label "Learning"

**Mastered:**
- Background: `color-mix(in oklab, var(--color-rune-500) 15%, var(--color-obsidian-800))`
- Border: 2px solid `--color-rune-500` (#d9a441)
- Glow: `box-shadow: 0 0 16px 4px color-mix(in oklab, var(--color-rune-500) 40%, transparent)`
- Badge: pill, `--color-rune-500` background, `--color-obsidian-950` text, label "Mastered"

### Difficulty Markers (D-04)

Three filled dots, bottom-left of node face. Dot size: 6px diameter, 4px gap.

| Difficulty | Filled dots | Color |
|------------|-------------|-------|
| Beginner | 1 of 3 | `--color-obsidian-600` (muted) for empty, `#e9e8ee` for filled |
| Intermediate | 2 of 3 | Same |
| Advanced | 3 of 3 | Same |

**Note:** `difficulty` lives on `NodeFrontmatterSchema` not `NodeSummarySchema`. The graph data projection must include `difficulty` — planner must decide whether to extend `NodeSummary` with a `GraphDisplayNode` projection or expose difficulty via a separate map. Document in ADR or CONTEXT.md extension.

## Edge Visual Language (D-03)

All edges are directed bezier curves. Arrow marker points to the dependent node (the one that depends on its prerequisite).

| State | Stroke color | Width | Opacity | Marker |
|-------|-------------|-------|---------|--------|
| At rest | `--color-obsidian-700` (#262631) | 1px | 50% | Arrow head, same color |
| Hovered chain | `--color-rune-400` (#ecc472) | 3px | 100% | Arrow head, rune-400 |

**Hover interaction:** When a node is hovered or selected, all edges on its full prerequisite chain (all ancestors) highlight simultaneously in `rune-400`. The highlight animates in with Motion `spring({ stiffness: 300, damping: 30 })` on entry and fades out with `duration: 0.2` on exit. No glow on edges — highlight is stroke-only to preserve three-gold differentiation.

Implementation note: React Flow `EdgeProps` + custom edge component with a Motion `motion.path` (or SVG `path` with Motion animation) is required for the animated highlight. The node hover state drives a React context or Zustand slice (planner's call) that the edge component reads.

## Guided-Pathway View Model (D-08, D-09)

**First load state:**
- Full graph is mounted in the DOM (all nodes present — satisfies memoization criterion 5)
- Non-pathway nodes: `opacity: 0.2`, pointer-events disabled
- Pathway nodes: `opacity: 1.0`, fully interactive
- Camera: `fitView` centered on pathway nodes, padding 0.2
- Pathway banner: fixed overlay showing pathway name + step count

**"Explore full map" action:**
- Reveals all nodes (opacity 1.0 on non-pathway nodes, pointer-events re-enabled)
- Camera: `fitView` on entire graph, padding 0.15, animated (Motion `animate` on `ReactFlow` instance via `useReactFlow`)
- Button disappears once activated; a "Back to pathway" button appears in the control bar

**Pathway data contract (D-10):**
- Location: `pathways/beginner-fundamentals.json`
- Zod-validated + CI referential-integrity check (every step node-id resolves to a real node)
- Shape: `{ id: string, title: string, subtitle: string, steps: string[] }` (ordered node IDs)

## Mobile Simplified Form (D-11, D-12)

**Breakpoint:** 768px (`md` in Tailwind v4 — `@media (min-width: 768px)`). Below 768px: graph canvas hidden, vertical scrollable list rendered.

**Mobile list layout:**
- Background: `--color-obsidian-900`
- Sticky header: "Beginner Fundamentals" in Space Grotesk 22px/600, 24px padding
- Section 1: Pathway nodes in pathway order, section label "Your Pathway"
- Section 2: All other nodes, section label "All Nodes"
- Section separator: 1px `--color-obsidian-700` divider + 8px vertical gap

**Mobile node card:**
- Height: 72px minimum
- Layout: `flex`, `align-items: center`, 16px horizontal padding
- Left: Type icon (20×20px lucide) + node title (15px/600, truncated 1 line)
- Right: Mastery badge (same spec as canvas badge, same visual states)
- Tap: `pointer-events: auto`, no navigation in Phase 2 (placeholder no-op; Phase 3 wires the detail panel)
- Touch target: full card row is the tap target (72px height satisfies 44px minimum)

## Animation Contract

All animation uses `motion/react` (not `framer-motion`). Respect `prefers-reduced-motion` (already handled globally in `app.css` — durations collapse to 0.001ms).

| Element | Animation | Spec |
|---------|-----------|------|
| Prerequisite-chain edges | Stroke color highlight on hover | Motion spring: `stiffness: 300, damping: 30` enter; `duration: 0.2` exit |
| Node entrance (initial load) | Fade-in by layer | `initial: { opacity: 0 }`, `animate: { opacity: 1 }`, delay by DAG layer (layer × 50ms, max 400ms total) |
| Pathway node entrance | Slight upward drift | `initial: { opacity: 0, y: 8 }`, `animate: { opacity: 1, y: 0 }`, spring |
| "Explore full map" reveal | Opacity transition on non-pathway nodes | `animate: { opacity: 1 }`, duration 0.4, ease "easeOut" |
| Camera fit-view to full map | React Flow `fitView` | Use `duration: 800` via `useReactFlow().fitView({ duration: 800 })` |
| Mastery state change (mock) | Glow pulse | `animate: { boxShadow: [...] }`, spring, plays once on state change |

No parallax, no scroll-linked animation, no continuous loops in Phase 2.
