<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compiled MDX → client render | `MDXContent` evaluates build-time-compiled JS on the client |
| citation `url` → `<a href>` (via mdxComponents) | Author URLs in MDX body links |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-01 | Tampering / XSS | mdxComponents `a` renderer | high | mitigate | The `a` component applies the http(s)-only allowlist before emitting an anchor (same rule as plan 03-06); blocks `javascript:`/`data:` (Pitfall 5). |
| T-3-08 | Tampering | `MDXContent` client eval | low | accept | MDX is repo-controlled/compiled at build (trusted authors); panel renders inside `<ClientOnly>` so eval never runs in SSR (Pitfall 4). Never pass user-generated MDX to the compiler. |
</threat_model>
