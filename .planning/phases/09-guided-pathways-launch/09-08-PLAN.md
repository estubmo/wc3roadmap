<objective>
Deliver the launch-readiness CI gate (CONT-04, PATH-02, CONT-05) as pure validators cloned from the validate-pathway pattern, wired into validate-content.ts behind a launch flag so the deploy is blocked until content lands without turning every in-progress PR red.

Planner decision (RESEARCH Open Question 2 / A3, and the D-11/D-12 "gate blocks launch" intent): the launch-gate checks are enforced only when the environment flag LAUNCH_GATE=1 is set. A dedicated `npm run validate:launch` sets it (used by the release/deploy gate); the always-on `npm run validate` (per-PR CI) does NOT, so the codebase stays mergeable while the parallel content workstream flips nodes to launch_ready. This resolves the tension where all 17 nodes start launch_ready:false (09-03) yet CI must remain green.

Purpose: Make "≥25 launch_ready nodes, all 8 pathway steps launch_ready, every launched node audited" a CI-enforced deploy invariant.
Output: pure validators + test + gated wiring + npm script.
</objective>
