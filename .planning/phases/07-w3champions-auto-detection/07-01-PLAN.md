<objective>
Create the project-owned MMR-tier registry (`src/lib/mmr-tiers.ts`) mirroring the `src/lib/patches.ts` singleton-registry pattern: a private ascending `_TIERS` array, a public readonly view, a `TIER_IDS` tuple for `z.enum()`, and two pure lookup helpers `tierForMmr(mmr)` and `tierIndex(id)`. This is the ordinal foundation the content schema, the pure detection function, and the fetch client all build on.

Purpose: Decouple the project's coarse skill-tier vocabulary from w3champions' season-fragile League names (RESEARCH Pitfall 5) so node-frontmatter thresholds never silently reinterpret when w3champions redraws league boundaries. One file to recalibrate, no migration anywhere else.
Output: `mmr-tiers.ts` + its Wave-0 unit test.
</objective>
