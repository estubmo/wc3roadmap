- Mastery button group: 4px gap between buttons (xs) — tighter spacing signals a segmented control, not independent buttons

## Phase 5 Interaction Contract

### MasteryControls Component

**Placement:** Top of NodePanelContent content area, inserted as the first element above "How to Apply in Your Next Game" section. Separated from content below by a 1px `obsidian-600` border-bottom with 16px margin-bottom.

**Layout (top to bottom):**
```
16px top padding
  "Mastery" — 13px / 600 / 1.2 (section label, color: #9998a8 subdued)
8px gap
  [Untouched] [In Progress] [Mastered]  — ToggleGroup, full-width flex row, 4px gap
8px gap
  "Marked manually · 2 hours ago"  — 12px / 400 / 1.4 (caption, only when record exists)
  OR "Sign in with Battle.net to save your progress across devices"  — 12px / 400 / 1.4 (signed-out users)
16px bottom padding
1px border-bottom: obsidian-600
```

**Button dimensions:**
- Width: `flex: 1` — equal thirds of the container minus 2×4px gaps
- Height: 36px
- Padding: 0 (let the flex height drive the height)
- Corner radius: 6px (matches shadcn new-york button radius)
- Font: 14px / 600 / 1.2 — Label size

**Button active states (mirrors node mastery visualization):**

| State | Bg | Text | Border |
|-------|-----|------|--------|
| Untouched (active) | `obsidian-800` | `#e9e8ee` | 1px `obsidian-600` |
| In Progress (active) | `rune-600` | `rune-300` | none |
| Mastered (active) | `rune-500` | `obsidian-950` | none |
| Any state (inactive) | `obsidian-800` | `#9998a8` (subdued) | 1px `obsidian-600` |
| Any state (hover, inactive) | `obsidian-700` | `#e9e8ee` | 1px `obsidian-600` |
| Any state (focus-visible) | current bg | current text | 2px `rune-400` ring, 2px offset |

**Accessibility:**
- `ToggleGroup` renders `role="group"` with `aria-label="Mastery state"` on the container
- Each `ToggleGroupItem` renders `aria-pressed` (active/inactive)
- Keyboard: Tab into group, Arrow keys cycle between items
- The section label "Mastery" is a `<span>` not `<h*>` — the panel's node title is the heading; this is a subdued label

### Optimistic Update Flow

1. User clicks a mastery state button
2. Immediate — `ToggleGroup` value switches to new state (visual update on button)
3. Immediate — Zustand `graph-store` updates `data.masteryState` for the affected node ID
4. Immediate — `GraphNode` re-renders for that single node (mastery border/glow/badge update)
5. Background — TanStack Mutation fires the `setNodeMastery` server function (authedServerFn)
6. On mutation success — no notification (silent)
7. On mutation error — revert Zustand store to previous state → revert `ToggleGroup` value → show error toast

**Only one node re-renders per mark** — no full graph refresh, no route reload (ROADMAP criterion 3).

### localStorage Merge Flow (PROG-03)

- Trigger: user signs in for the first time with localStorage progress records present
- Behavior: fill-gaps merge (D-07) — server records win; localStorage fills only nodes the server has no record for
- After merge: localStorage cleared
- Notification: if any records were merged → show merge toast (5s auto-dismiss)
- Notification: if no localStorage records existed → silent (no toast)
- No conflict-surfacing UI — server always wins on conflict (D-07)

### Signed-Out State

Signed-out users CAN mark mastery (PROG-03 supports pre-login progress in localStorage):
- `MasteryControls` renders identically
- State writes go to localStorage instead of server function
- Source attribution line shows: "Sign in with Battle.net to save your progress across devices"
- No "Sign In" button embedded in the controls — the existing SiteHeader Sign In CTA handles auth

### Toast Specifications (Sonner)

**Position:** bottom-right on desktop, bottom-center on mobile

**Error toast:**
```
[!] Couldn't save your progress
    Your selection has been reverted. Please try again.
    [Retry]  ×
```
- Duration: persistent until dismissed or retry attempted
- Style: `obsidian-800` bg, `obsidian-600` border, 4px `rune-600` accent left-border

**Merge toast:**
```
[✓] Progress synced
    Your progress from your previous session has been synced.
                                                              ×
```
- Duration: 5000ms auto-dismiss
- Style: `obsidian-800` bg, `obsidian-600` border, 4px `rune-500` accent left-border

## Animation Contract

| Element | Animation | Spec | Library |
|---------|-----------|------|---------|
| Mastery button active state switch | Instant (no animation) | 0ms — optimistic feel; snappiness signals instant save | — |
| GraphNode mastery state change | CSS transition on bg + border + box-shadow | `transition: background-color 150ms ease, border-color 150ms ease, box-shadow 150ms ease` | CSS only |
| MasteryBadge enter (untouched → any) | Fade in | 150ms ease-out opacity 0→1 | Motion `AnimatePresence` |
| MasteryBadge exit (any → untouched) | Fade out | 100ms ease-in opacity 1→0 | Motion `AnimatePresence` |
| Error toast enter/exit | Managed by Sonner | Sonner defaults (slide + fade) | Sonner |
| Merge toast enter/exit | Managed by Sonner | Sonner defaults (slide + fade) | Sonner |

**`prefers-reduced-motion` rule:** `src/styles/app.css` already contains a global `prefers-reduced-motion` override that collapses all animation/transition durations to 0.001ms. No per-component reduced-motion handling needed.
