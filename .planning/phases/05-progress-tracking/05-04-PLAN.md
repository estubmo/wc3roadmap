<objective>
Build the progress persistence deep module: three `authedServerFn` server functions — `getUserProgress` (read all rows for the principal), `setNodeMastery` (upsert one node's state), and `mergeProgressOnSignIn` (one-time fill-gaps merge of localStorage payload). All are principal-keyed by construction (D-06): no `userId` input channel exists. `source` is hardcoded `"manual"` (D-04) and `patchId` is stamped from `CURRENT_PATCH.id` (D-05) — never read from client input.

Purpose: PROG-01/PROG-02 (per-user server-side persistence) and the server half of PROG-03 (merge). This is the auditable seam where all progress writes happen; the simple `authedServerFn` interface hides upsert + patch-stamping + fill-gaps complexity (deep-module discipline).
Output: `src/server/progress.ts` + unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server fn | masteryState/nodeId/merge payload cross from untrusted client into authedServerFn handlers |
| server fn → database | upsert/insert/select across Drizzle+Neon |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04a | Tampering / Info Disclosure | progress server fns (IDOR) | high | mitigate | authedServerFn principal-keyed by construction (D-06, ADR-007); no userId input channel — cross-user read/write impossible |
| T-05-04b | Tampering | crafted masteryState | medium | mitigate | MasteryStateSchema Zod parse before any DB write rejects values outside the three-state enum |
| T-05-04c | Tampering | crafted patchId / source | medium | mitigate | server stamps CURRENT_PATCH.id and hardcodes source "manual"; neither is read from client input |
| T-05-04d | Tampering | localStorage poisoning via merge | medium | mitigate | merge handler validates every record's masteryState via MasteryStateSchema; invalid records skipped; fill-gaps means server records never overwritten (D-07) |
| T-05-04e | Spoofing | unauthenticated call | high | mitigate | authMiddleware throws Unauthorized before handler runs (D-11) — handler never executes without a session |
</threat_model>
