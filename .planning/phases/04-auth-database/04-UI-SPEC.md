## Typography

All weights use Outfit Variable (body/UI) or Space Grotesk Variable (heading/display).
Exactly 2 weights used in this phase: regular (400) and semibold (600).

Weight-400 (regular): body text AND all interactive labels (region option buttons, dropdown items, BattleTag display chip).
Weight-600 (semibold): headings AND the primary "Sign in with Battle.net" CTA button text.

shadcn override required: shadcn Button defaults to `font-medium` (500). Override with `font-semibold` (600) on the primary Sign-in CTA and `font-normal` (400) on secondary interactive labels (region option buttons). Do not rely on shadcn's 500 default anywhere in this phase.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body / Interactive labels | 14px | 400 | 1.5 | Outfit Variable | Body text; dropdown items; region selector description; region option button labels |
| BattleTag chip | 13px | 400 | 1.4 | Outfit Variable | Signed-in header BattleTag display |
| Heading / Primary CTA | 16px | 600 | 1.25 | Space Grotesk Variable | Region selector dialog title; "Sign in with Battle.net" button text |

## Interaction Patterns

### Sign-in Flow

```
[User] clicks "Sign in with Battle.net" (top-right header)
  → <RegionSelector> dialog opens
  → [User] clicks a region button (Americas / Europe / Korea)
  → Region stored in sessionStorage key "bnet_region"
  → better-auth OAuth redirect initiated: authClient.signIn.oauth2({providerId: "battlenet", ...})
  → [User] authenticates on Battle.net
  → OAuth callback returns to app
  → Session cookie set; user record written to Neon DB
  → Header updates: SignInButton → UserDropdown (BattleTag + avatar)
  → No page reload — session state drives header re-render
```

### Sign-out Flow

```
[User] clicks UserDropdown trigger (avatar + BattleTag chip)
  → Dropdown opens
  → [User] clicks "Sign out"
  → authClient.signOut() called
  → Session cookie cleared
  → Header updates: UserDropdown → SignInButton
  → No confirmation dialog — no data at risk in Phase 4
  → No page reload
```

### Session Persistence

```
[User] refreshes browser
  → Session cookie present (30-day rolling; always-persistent)
  → better-auth resolves session server-side
  → Header renders UserDropdown immediately (no flash of sign-in state)
  → SSR-safe: session state derived from cookie, not client localStorage
```

### OAuth Error Handling

```
[OAuth fails — network error or Battle.net returns error]
  → Header renders SignInButton (user remains signed out)
  → Error toast or inline header message (14px / 400):
    "Sign-in failed — check your connection and try again."
  → No blocking modal — user can continue using the app signed-out (D-03)
```

## Accessibility Notes

| Element | Requirement |
|---------|-------------|
| Sign-in CTA button | `aria-label="Sign in with Battle.net"` (redundant but explicit for screen readers) |
| Region selector dialog | `role="dialog"`, `aria-labelledby` pointing to dialog title; focus trap active while open |
| Region option buttons | Each button is a native `<button>` — no div-click shenanigans |
| UserDropdown trigger | `aria-haspopup="menu"`, `aria-expanded` toggled; keyboard: Enter/Space opens |
| Avatar | `alt={battleTag}` on the img; AvatarFallback for when image fails to load |
| Header | `<header role="banner">` wrapping `<SiteHeader>` |
| Focus order | Sign-in button reachable by Tab from page start; region dialog traps focus |
| Touch targets | All interactive elements ≥ 44px height (region options: 44px; sign-in button: 40px — acceptable for desktop-primary app per GRAPH-05) |
