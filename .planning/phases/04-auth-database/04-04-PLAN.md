<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client request body/params -> server function | Client-supplied userId is untrusted and must be ignored |
| session cookie -> principal | The session principal is the ONLY trusted identity source |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-04a | Elevation of Privilege | forged userId in request | critical | mitigate | D-12: user-data server fns ignore client input entirely and key by context.principal.id; getUserProfile has no userId parameter — cross-user access is impossible by construction. Regression-locked by the D-13 test. |
| T-04-04b | Spoofing | missing/forged session | high | mitigate | D-11: authMiddleware throws 401 before the handler runs when getSession returns null; next() never fires (test-asserted) |
| T-04-04c | Elevation of Privilege | reliance on beforeLoad instead of middleware | high | mitigate | Pitfall 7: every user-data server fn applies authMiddleware (the HTTP-endpoint boundary); beforeLoad is UX-only. Convention documented in plan 04-07 ADR. |
</threat_model>
