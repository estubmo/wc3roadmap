<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client component -> auth surface | Components may touch only the client-safe auth-client, never server auth.ts |
| sessionStorage gateway -> OAuth flow | Region is user-chosen client state carried into the OAuth redirect |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05a | Information Disclosure | server auth import in client component | high | mitigate | Components import only from #/lib/auth-client; verify-gate greps for forbidden #/lib/auth import |
| T-04-05b | Tampering | gateway value from sessionStorage | low | accept | Gateway is non-security display/routing metadata; a user editing their own region only affects their own w3champions lookup key (Phase 7), no cross-user impact |
| T-04-05c | Information Disclosure | DiceBear avatar request leaks BattleTag to a third party | low | accept | BattleTag is already a public identity; the generated-avatar fallback can be swapped for a local CSS initials avatar (Pitfall 2 / A7) if desired without changing the contract |
</threat_model>
