<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/auth/$ | Untrusted OAuth callback + session requests cross here |
| Battle.net OAuth provider -> app | External identity provider; token + profile must be validated |
| server bundle vs client bundle | auth.ts secrets must not cross into the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-03a | Spoofing | OAuth callback | high | mitigate | better-auth generic-OAuth manages PKCE + `state` param (CSRF) and validates token exchange; we do not hand-roll the flow |
| T-04-03b | Elevation of Privilege | session token | high | mitigate | better-auth issues httpOnly, SameSite=Lax cookie, signs cookieCache, rotates token on each OAuth callback (session fixation); BETTER_AUTH_SECRET >= 32 chars (V6) |
| T-04-03c | Information Disclosure | auth.ts secrets in client bundle | critical | mitigate | auth.ts is server-only; verify-gate asserts auth-client.ts never imports #/lib/auth; BNET_CLIENT_SECRET/BETTER_AUTH_SECRET are server env vars only |
| T-04-03d | Tampering | open redirect on callbackURL | medium | mitigate | better-auth validates callbackURL against its allowlist; callbackURL is app-controlled ("/"), never taken from arbitrary client input |
</threat_model>
