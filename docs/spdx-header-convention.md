# SPDX Header Convention

Every hand-authored source file in this repository must carry an SPDX
license identifier on **line 1** — the very first line of the file, before
any imports, `use strict`, or other content (see RESEARCH Pitfall 5 and
[SPDX specification §8.1](https://spdx.github.io/spdx-spec/v2.3/file-information/)).

## Required header (line 1)

```
// SPDX-License-Identifier: GPL-3.0-or-later
```

Optionally followed on line 2 by a copyright notice:

```
// Copyright (C) 2026 WC3 Roadmap contributors
```

The SPDX variant is **GPL-3.0-or-later**, confirmed by the human decision
recorded in [ADR 004](adr/004-gpl3-licensing.md) before the `LICENSE` file
was committed. This is the identifier used in `package.json` `"license"` and
in the root `LICENSE` file.

## File globs this convention applies to

| Glob | Description |
|------|-------------|
| `src/**/*.ts` | TypeScript source modules |
| `src/**/*.tsx` | TypeScript React components and routes |
| `scripts/**/*.ts` | Build and utility scripts |
| `content-collections.ts` | Content-collections Vite configuration |
| `vite.config.ts` | Vite bundler configuration |
| `vitest.config.ts` | Vitest test configuration |

## Exclusions

The following file types are **exempt** from the SPDX header requirement:

| File / Glob | Reason |
|-------------|--------|
| `src/routeTree.gen.ts` | Auto-generated by `@tanstack/router-cli`; regenerated on every `tsr generate` run — hand-edits (including headers) are overwritten |
| `*.json` | JSON does not support comments |
| `*.md` | Markdown documentation — license stated in repo root `LICENSE` |
| `*.css` | Generated or static stylesheets — no comment header convention enforced |
| `content/**` | MDX / JSON content files — covered by repo `LICENSE` |
| `.planning/**` | GSD planning artifacts — not distributed source code |

## Enforcement

- **Pre-commit:** Run `grep -rL "SPDX-License-Identifier: GPL-3.0-or-later" src/ scripts/ --include="*.ts" --include="*.tsx"` to find files missing the header. This should return no output for a compliant repo.
- **CI (planned):** Add a lint step that runs the above grep and fails the build if any hand-authored TypeScript file lacks the SPDX header.
- **New files:** When adding a new `.ts` or `.tsx` file under the globs listed above, place `// SPDX-License-Identifier: GPL-3.0-or-later` as the very first line before any other content.

## Changing the variant

If the GPL-3.0 variant is ever revised (e.g., if wc3v turns out to be
GPL-3.0-only), the mechanical sweep is:

```bash
# Preview
grep -rn "SPDX-License-Identifier: GPL-3.0-or-later" src/ scripts/ content-collections.ts vite.config.ts vitest.config.ts

# Apply (adjust old/new variant as needed)
find src/ scripts/ -name "*.ts" -o -name "*.tsx" | \
  xargs sed -i 's|GPL-3.0-or-later|GPL-3.0-only|g'
sed -i 's|GPL-3.0-or-later|GPL-3.0-only|g' content-collections.ts vite.config.ts vitest.config.ts
```

Update `package.json` `"license"` and the root `LICENSE` accordingly, then
record the change in a new ADR.
